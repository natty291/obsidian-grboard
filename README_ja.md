# GRBOARD — Obsidian プラグイン

**GRBOARD(Go/Renju(Gomoku) BOARD)** は、囲碁や連珠・五目並べの盤面画像から石を自動検出し、SGF形式に変換する Obsidian プラグインです。また、変換した SGF をノートに直接埋め込んで閲覧・編集・対局ができるフル機能の SGF ビューアも提供します。

## はじめに

[obsidian-goboard-viewer](https://github.com/j2masamitu/obsidian-goboard-viewer)にてobsidian で囲碁の盤面をインタラクティブに操作できることに驚き、連珠にも適用できるのではないか。詰連珠を楽しむために、画像から問題の盤面を再現したいと思い、本プラグインを興味のあったバイブコーディングにて作成し、￥詰連珠を楽しむために必要な機能は実現できたのではないかと思います。盤面の読み込み機能に限れば、囲碁、連珠(五目並べ)に差はなく、どちらのユーザーでも楽しんでいただけるのではないかと思います。

盤の描画/操作に関しては、ほとんどの部分を元のプラグインから流用させてもらっています。素晴らしいプラグインをありがとうございます。

---

## 概要

GRBOARD は、棋譜画像から盤面の局面を自動で読み取り、Obsidian 内でそのまま検討・編集・再生できる SGF ファイルに変換するというニーズに応えます。(読み取れない画像もあります)

局面を SGF として取り込んだあとは、シンプルなフェンスコードブロックを貼るだけで Vault のどこにでも盤面をインタラクティブに表示できます。

## インストール方法

### 手動インストール

1. [Releasesページ](https://github.com/natty291/obsidian-grboard/releases)から最新版をダウンロード
2. ファイルをObsidian vaultの`.obsidian/plugins/grboard/`ディレクトリに展開
3. **設定 → コミュニティプラグイン** でプラグインを有効にします。
4. 左サイドバーのリボンに **Board Detect** のアイコン（スキャン記号）が表示されます。

### ソースからビルド

1. このリポジトリをクローン
2. 依存関係をインストール: `npm install`
3. プラグインをビルド: `npm run build`
4. 以下のファイルを`.obsidian/plugins/grboard/`にコピー:
   - `manifest.json`
   - `main.js`
   - `styles.css`
5. **設定 → コミュニティプラグイン** でプラグインを有効にします。
6. 左サイドバーのリボンに **Board Detect** のアイコン（スキャン記号）が表示されます。

## Board Detect ビュー

リボンアイコンをクリックするか、コマンドパレットから **「Open Board Detect view」** を実行してビューを開きます。

---

### 起動：モードを選択する

ビューを開くと、最初に 2 つの選択肢が表示されます。

| ボタン | 説明 |
|---|---|
| 📂 **画像を読み込む** | 盤面の写真やダイアグラム画像をデバイスから開く |
| 🖼 **デフォルト画像生成** | テスト・キャリブレーション用のサンプルボードを生成する |

---

### ステップ 1 — 画像の読み込みまたはサンプル生成

**画像を読み込む**
ボタンをクリックしてファイルピッカーを開き、画像ファイル（JPEG・PNG など）を選択します。メインビューに表示されるドロップゾーンに画像をドラッグ＆ドロップすることもできます。

**デフォルト画像生成**
ボタンをクリックするとオプションパネルが展開されます。

| 項目 | 説明 |
|---|---|
| 横（列） | 横方向の交点数（1〜19） |
| 縦（行） | 縦方向の交点数（1〜19） |
| 石の数 | ランダムに配置する石の総数 |

**▶ 生成して開始** をクリックするとボードが描画されます。石は黒白交互にランダム配置され、正解データが内部に保存されるため、検出後に精度を照合できます。

---

### ステップ 2 — 盤面サイズの設定

**盤面サイズ** カードで、画像に写っている盤面の列数と行数を入力します。この値をもとに、検出エンジンが画像をグリッド状の交点に分割します。

- 設定範囲：縦・横ともに 1〜19
- 長方形の盤面（例：13 × 9）も対応しています

---

### ステップ 3 — 余白のカット

**余白カット** カードでは、盤面以外の部分（テーブルの端・枠・背景など）を上下左右それぞれトリミングできます。

4 辺（上・下・左・右）それぞれに **スライダーと数値入力欄** があり、互いに連動しています。値は元画像のピクセル数で指定します。

ソース画像キャンバス上には、現在のカット領域を示す **黄色の破線矩形のオーバーレイ** がリアルタイムで表示されます。コントロールの下には余白の割合を視覚的に示す小さな **ダイアグラム** も表示されます。

コントロールの上部に現在の画像サイズ（ピクセル）が表示されます。

---

### ステップ 4 — 検出パラメータの調整

**判定パラメータ** カードには 3 つのスライダーがあります。

| パラメータ | 範囲 | デフォルト | 説明 |
|---|---|---|---|
| **黒閾値** | 30〜150 | 80 | この輝度値を下回るピクセルを「暗い」とみなします。交点の平均輝度がこの値を下回ると黒石と判定します。 |
| **白閾値** | 100〜240 | 170 | この輝度値を超える交点を白石の候補とします。 |
| **輪郭スコア** | 5〜90 % | 40 % | 白石候補の円周上に検出された暗いピクセルの割合の最低値。これを下回ると空点と判定します。 |

> **注意：黒閾値は白石の輪郭スコア計算にも影響します。**
> 白石の判定では円周上の「暗いピクセル」を数えますが、その「暗い」の基準として黒閾値の 1.8 倍の値が内部的に使われています。そのため黒閾値を変更すると、黒石の判定だけでなく白石の輪郭スコアの感度も同時に変化します。黒閾値を上げると白石が検出されやすくなり、下げると検出されにくくなります。

**検出の仕組み（概要）**

1. 画像を盤面領域にトリミングしてリサイズします。
2. 各交点について、円形サンプリング領域の平均輝度を計算します。
3. 輝度 ＜ 黒閾値 → **黒石**
4. 輝度 ＞ 白閾値 → 円周輪郭スキャンを実行（暗いピクセルの基準 = 黒閾値 × 1.8）。輪郭スコア ≥ 閾値 → **白石**、それ以外 → **空点**
5. それ以外 → **空点**

**マーカーサイズ** コントロール（検出実行ボタンの下）では、検出結果のキャンバス上に重ねて表示する色付きドットの半径を、グリッド 1 マスに対する割合（5〜100 %）で調整します。

---

### ステップ 5 — 検出の実行

**▶ 検出実行** をクリックすると、検出パイプラインが実行されます。

- カット後のボード画像がコントロールの下に表示されます。
- 各交点に色付きのマーカーがオーバーレイされます：
  - **シアン / 青** — 黒石と検出
  - **赤 / オレンジ** — 白石と検出
  - マーカーなし — 空点
- **統計** カードに黒石数・白石数・空点数・合計交点数が表示されます。
- サンプルボードを使用した場合、**精度評価レポート** が表示され、正解した交点数と誤検出の一覧が確認できます。

検出が完了すると、**ボード展開** カードと **SGF 出力** カードが利用可能になります。

---

### ステップ 5.5 — 検出結果の補正

自動検出の結果が正しくない交点がある場合、**✏ 補正** ボタンで個別に修正できます。

#### 補正モードの起動

**▶ 検出実行** ボタンの右にある **✏ 補正** ボタンをクリックすると補正モードが開始されます。ボタンがオレンジ色に点滅し、カット後のボード画像上でカーソルが十字に変わります。

> 補正ボタンを再度クリックすると、すべての補正がリセットされた状態で補正モードが再起動します。

#### 交点をタップして状態を変更する

補正モード中は、カット後の画像上の交点をタップするたびに状態がトグルします。自動検出の結果をスキップして次の状態から始まります。

| 自動検出の結果 | 1 回目 | 2 回目 | 3 回目 |
|---|---|---|---|
| 黒石 | 白石 | 石なし | 解除（自動に戻る） |
| 白石 | 石なし | 黒石 | 解除（自動に戻る） |
| 石なし | 黒石 | 白石 | 解除（自動に戻る） |

#### 補正の表示

補正した交点には、自動検出マーカーと同じスタイルで状態が上書き表示されます。

| 補正後の状態 | 表示 |
|---|---|
| 黒石 | 黒石マーカー色の塗り潰し円 |
| 白石 | 白石マーカー色の塗り潰し円 |
| 石なし | 赤い ❌ |

#### 補正モードの終了

**✏ 補正** ボタンを再度クリックするか、**▶ 検出実行** を再実行すると補正モードが終了します。別の画像を読み込んだ場合も補正はリセットされます。

> **注意：** 補正した内容は SGF エクスポートおよびボード展開の結果にも反映されます。

---

### ステップ 5.6 — 検出画像への碁盤オーバーレイ

カット後の検出画像（マーカー付き）の上に、碁盤グラフィック（石のイラスト）を半透明で重ねて表示できます。補正や設定変更の結果が実際の盤面写真と一致しているかを最終確認するための機能です。

#### オーバーレイの起動

**✏ 補正** ボタンの下にある **🔍 実画像を重ねる** ボタンをクリックするとオーバーレイが開始されます。ボタンが青紫色に変わり、カット後の画像の上に碁盤グラフィックが半透明で重なって表示されます。再度クリックすると非表示になります。

#### 透明度の調整

ボタンの右にある **スライダー**（10〜90%、5% 刻み）で碁盤グラフィックの透明度を調整できます。スライダーを動かすと即座に反映されます。

- 値を小さくする → 碁盤グラフィックが薄くなり、実画像が見やすくなる
- 値を大きくする → 碁盤グラフィックが濃くなり、石の位置との対応が確認しやすくなる

#### 補正との連動

補正モードで交点の状態を変更すると、オーバーレイも自動的に更新されます。実画像の石の位置と碁盤グラフィック上の石の位置がずれていないかをリアルタイムで確認できます。

> 別の画像を読み込んだ場合や再設定を行った場合は、オーバーレイは自動的にOFFになります。

---

### ステップ 6 — ボード展開（オプション）

**ボード展開** カードは、部分盤面（コーナー図や辺の図など）を検出した場合に、正しい位置に配置した全体の盤面に変換するための機能です。

| 項目 | 説明 |
|---|---|
| **出力 横** | 出力する全体ボードの列数（1〜19） |
| **出力 縦** | 出力する全体ボードの行数（1〜19） |
| **オフセット X** | 検出ボードの左端を配置する列インデックス（0 始まり） |
| **オフセット Y** | 検出ボードの上端を配置する行インデックス（0 始まり） |

**▶ 展開実行** をクリックすると適用されます。結果は **展開結果** カードにサマリーとキャンバスで表示され、以降の SGF エクスポートにはこの展開後のボードが使用されます。

このステップをスキップして SGF エクスポートに進もうとすると、先に展開を実行するよう促されます（オフセットが不要な場合でも、同じサイズで展開実行してください）。

---

### ステップ 7 — SGF のエクスポート

検出（および展開）が完了すると **SGF 出力** カードが表示されます。以下の項目を設定します。

| 項目 | 選択肢 | 説明 |
|---|---|---|
| **初手** | — / 黒先 / 白先 | SGF ルートノードの `PL` プロパティを設定 |
| **ゲームモード** | 囲碁 / 連珠・五目 | `GM[1]`（囲碁）または `GM[4]`（連珠/五目）を設定 |
| **ルール** | 自由入力 | `RU` プロパティを設定（例："Puzzle info"）。省略可 |

**▶ SGF 作成** をクリックすると SGF 文字列が生成され、読み取り専用のテキストエリアに ` ```grboard ` コードブロック形式で表示されます。

ここから以下の操作が可能です：

- **📋 コピー** — コードブロックごとクリップボードにコピー（` ```grboard ` フェンスを含む）
- **💾 Vault に保存** — コードブロックなしの生の SGF を、タイムスタンプ付きの `.sgf` ファイルとして Vault ルートに保存（例：`grb-20260620T143000.sgf`）

---

### リセット

**再設定** ボタン（赤枠）をクリックすると確認ダイアログが表示されます。確認すると、すべての検出結果・読み込み画像・SGF 出力がクリアされ、起動画面に戻ります。

## SGF ビューア（grboard コードブロック）

ノートの任意の場所に `grboard` フェンスコードブロックを貼り付けると、インタラクティブな盤面ビューアが埋め込まれます。

````markdown
```grboard
(;GM[1]FF[4]SZ[19]PL[B]
AB[dd][pp][dp][pd]
AW[qq][pq]
;B[qd];W[dc];B[ce];W[ed])
```
````

ビューアはモード選択ドロップダウンの下に描画されます。ドロップダウンで **参照 / Play / 編集** の 3 モードを切り替えられます。

SGF に対局情報（棋士名・段位・日付・棋戦・結果・コミ・ルール）が含まれている場合、盤面の上に表示されます。`PL` プロパティが設定されている場合は初手の色を示すバナーも表示されます。

右上の 📷 **画像保存ボタン** で現在の盤面を PNG ファイルとしてエクスポートできます。ボタンをクリックすると確認ダイアログが表示され、以下のオプションを選択できます。

| オプション | 説明 |
|---|---|
| **マーカーを含める** | 最終手マーカー・SGF シンボルマーカー・手順番号をPNGに出力する（デフォルト：ON） |
| **座標を含める** | 盤面の周囲に座標ラベル（列：A〜T、I除く／行：1〜N）を付けてエクスポートする（デフォルト：OFF） |

保存ファイル名はタイムスタンプ形式です（例：`grb-20260620T143000.png`）。

---

### 参照モード

棋譜を読んで手順をたどるためのデフォルトモードです。

**ナビゲーションボタン**

| ボタン | 動作 |
|---|---|
| ⏮ 最初 | 開始局面（0 手目）にジャンプ |
| ◀ 前 | 1 手戻る |
| 次 ▶ | 1 手進む |
| ⏭ 最後 | 最終手にジャンプ |

**最終手マーカー**

最後に打たれた石に色付きの丸が表示されます（デフォルト：緑）。

**SGF マーカー**

SGF に定義されたシンボル（三角 `TR`・四角 `SQ`・丸 `CR`・バツ `MA`・テキストラベル `LB`）が盤面上に表示されます。

**変化（バリエーション）**

現在の局面に分岐がある場合、**変化を選択** パネルが表示され、各変化に A・B・C… とラベルが付いたボタンで切り替えられます。

**ノードコメント**

現在の SGF ノードにコメント（`C` プロパティ）が含まれている場合、コントロールの下にイタリック体で表示されます。

**自動再生**

ナビゲーションボタンの下に自動再生コントロールが表示されます（参照モードのみ）。ドロップダウンで再生速度（1 手あたり 0.5 秒・1 秒・2 秒・3 秒）を選択し、再生ボタンを押すと自動で手順が進みます。

---

### Play モード

Play モードでは盤面をクリックして実際に手を打つことができます。

- 現在の手番が **● 黒番** または **○ 白番** として表示されます。
- すでに石がある交点をクリックしても無効です。
- 手番は自動で交互に切り替わります。初手の色は SGF の `PL` プロパティで決まり、指定がない場合は黒番から始まります。
- ナビゲーションボタン（⏮ ◀ ▶ ⏭）で打った手を前後に移動できます。
- 途中まで戻ってから新しい手を打つと、それ以降の手が破棄されます。

**手順表示**

**手順表示** チェックボックスをオンにすると、これまでに打った各石に手順番号のラベルが表示されます。取られた石は除外されます。オフにすると最後に打った石にのみ丸マーカーが表示されます。

**📋 コピー** ボタンで、現在の対局状態（元 SGF の配置石と今回打った手をすべて含む）の SGF 文字列を生成してクリップボードにコピーできます。

> **注意：** 石の取り上げは囲碁モード（`GM[1]`）のみ有効です。連珠/五目モード（`GM[4]`）では無効になります。

---

### 編集モード

編集モードでは、ノート内のコードブロックの SGF コンテンツを直接編集できます。

**クリックモード選択**

ドロップダウンで、交点をクリックしたときの動作を設定します。

| モード | 動作 |
|---|---|
| **Move** | 次の手を打つ（手番交互、囲碁モードでは取り上げあり） |
| **Black Stone** | 黒石のセットアップ石を配置（`AB`） |
| **White Stone** | 白石のセットアップ石を配置（`AW`） |
| **Triangle** | 三角マーカーを追加（`TR`） |
| **Square** | 四角マーカーを追加（`SQ`） |
| **Circle** | 丸マーカーを追加（`CR`） |
| **Mark (X)** | バツマーカーを追加（`MA`） |
| **Label** | テキストラベルを追加（`LB`）— テキスト入力欄が表示され、最大 3 文字まで入力可 |

すでに同じマーカーがある交点をクリックすると削除されます（トグル動作）。

**コメント編集**

現在の局面のコメント（`C` プロパティ）を表示・編集するテキストエリアが表示されます。**💾 save comment** をクリックして適用します。

**対局情報の編集**

標準的な SGF 対局情報プロパティをすべて編集できるグリッド形式の入力欄が表示されます。

- 黒番棋士名・段位（`PB`・`BR`）
- 白番棋士名・段位（`PW`・`WR`）
- 棋譜名（`GN`）・棋戦名（`EV`）・ラウンド（`RO`）
- 日付（`DT`）・対局場所（`PC`）
- コミ（`KM`）・ハンデ（`HA`）・結果（`RE`）・ルール（`RU`）
- 初手（`PL`）

**💾 save game info** をクリックして適用すると、SGF 出力テキストエリアが即座に更新されます。

**🗑 delete from here** ボタンで、現在の局面以降のすべての手を削除します。残ったノードから手の木が再構築されます。

**ファイルへの書き戻し**

**💾 Write to note** ボタン（コードブロックにソースコンテキストがある場合に表示）をクリックすると、元のノートファイルの `grboard` コードブロックが更新後の SGF で上書きされます。保存後、ビューは自動的にそのブロックまでスクロールします。

SGF 出力テキストエリア（読み取り専用）には常に現在の SGF ツリーの状態が反映されており、手動でコピーすることもできます。

---

### コードブロックオプション

コードブロックのインフォ文字列（` ```grboard ` の後ろ）でオプションを指定できます。

```
```grboard bgcolor="#c8a84b"
```

| オプション | 説明 |
|---|---|
| `bgcolor="#RRGGBB"` または `bgcolor="(r,g,b)"` | この盤面だけに適用する背景色。設定の「盤面の背景色」より優先されます。 |

Vault 内の `.sgf` ファイルをファイルエクスプローラでクリックすると、プラグインが独自のビューでファイルを開きます。参照 / Play / 編集の各モードを切り替えるドロップダウン付きのインタラクティブな盤面が表示されます。

## ノートへの SGF 埋め込み

Obsidian の標準的な埋め込み記法を使って、Vault 内の `.sgf` ファイルをノートにインラインで埋め込めます。

```markdown
![[my-game.sgf]]
```

プラグインが埋め込みを自動的に検知し、インタラクティブな盤面ビューアに置き換えます。

### 表示内容

embed は **参照モード固定**で表示されます。`grboard` コードブロックと同様に以下が表示されます。

| 要素 | 内容 |
|---|---|
| 盤面 | 座標ラベル付きのインタラクティブな盤 |
| 最終手マーカー | 最後に打たれた石に色付きの丸を表示 |
| SGF マーカー | SGF に定義されたシンボル（△ □ ○ × テキストラベル）|
| 対局情報 | SGF に含まれる棋士名・棋戦・日付・結果等 |
| ナビゲーション | ⏮ 最初 / ◀ 前 / 次 ▶ / ⏭ 最後 |
| Move 情報 | `Move: N / 総手数` |
| 自動再生 | ▶ auto play（速度調整可能） |

> **注意：** embed では SGF の書き換えを防ぐため、編集モード・Play モードへの切り替えおよび「write to note」ボタンは表示されません。また 📷 PNG 保存ボタンも表示されません

### 基本的な使い方

```markdown
![[my-game.sgf]]
```

### 指定手数から表示する

`|move=N` を付けると、N 手目の局面を初期状態として表示します。そこからナビゲーションボタンで前後に移動できます。

```markdown
![[my-game.sgf|move=5]]
```

上の例では 5 手目の局面が初期表示となり、⏮ 最初ボタンで 0 手目（初期配置）に戻ることができます。

## 設定

**設定 → Board Detect** を開いてプラグインを設定します。

### 言語

| 選択肢 | 説明 |
|---|---|
| English | UI を英語で表示 |
| 日本語 | UI を日本語で表示 |

言語を変更すると、プラグインビュー・設定画面・コマンドパレットのエントリが即座に更新されます。

### 表示設定

すべてのカラーピッカーに **デフォルトに戻す** ボタンがあります。

| 設定項目 | デフォルト | 説明 |
|---|---|---|
| **盤面の背景色** | `#d4a843`（木目調） | 検出結果ボードおよび SGF ビューアの盤面背景色 |
| **黒石マーカー色** | シアン `rgba(0,200,255,0.8)` | 検出結果で黒石に重ねるドットの色 |
| **白石マーカー色** | 赤 `rgba(255,80,80,0.8)` | 検出結果で白石に重ねるドットの色 |
| **最終手マーカー色** | `#00aa00`（緑） | 最後に打たれた石に表示する丸の色 |
| **シンボルマーカー色（石上）** | `#ff0000`（赤） | 石の上に表示する SGF シンボルマーカー（△ □ ○ ×）の色 |
| **シンボルマーカー色（空点上）** | `#222222`（ほぼ黒） | 空点の上に表示する SGF シンボルマーカーの色 |
| **手順番号色（黒石上）** | `#ffffff`（白） | 黒石上の手順番号ラベルの文字色 |
| **手順番号色（白石上）** | `#000000`（黒） | 白石上の手順番号ラベルの文字色 |

Board Detect ビュー内の **マーカーサイズ** コントロールでは、検出オーバーレイのドットサイズをグリッド 1 マスに対する割合（5〜100 %）で調整でき、値は自動的に設定として保存されます。

## ナビゲーションボタンの並び順をカスタマイズする

SGF ビューアのナビゲーションボタン（⏮ 最初 / ◀ 前 / 次 ▶ / ⏭ 最後）の表示順は、`styles.css` の CSS `order` プロパティで変更できます。

### デフォルトの設定
** ⏮ ◀ ▶ ⏭ **
```css
.grboard-btn-group .grboard-btn-first { order: 1; }
.grboard-btn-group .grboard-btn-prev  { order: 2; }
.grboard-btn-group .grboard-btn-next  { order: 3; }
.grboard-btn-group .grboard-btn-last  { order: 4; }
```

`order` の値が小さいほど左に表示されます。4 つのボタンにそれぞれ異なる値を割り当てることで、任意の順番に並び替えられます。

### カスタマイズ例

**◀ ▶ ⏮ ⏭ — Prev / Next を左に寄せる**

```css
.grboard-btn-group .grboard-btn-first { order: 3; }
.grboard-btn-group .grboard-btn-prev  { order: 1; }
.grboard-btn-group .grboard-btn-next  { order: 2; }
.grboard-btn-group .grboard-btn-last  { order: 4; }
```
